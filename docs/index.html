01 · 真实终端

嵌入的是真实 CLI，不是模拟器

点一下侧栏芯片，对应 CLI 的原生 TUI 就作为独立窗口出现在画布上——拖动、八向缩放、贴边吸附半屏、推角四分屏、一键平铺。每个终端永久带着创建时的项目标识，跨项目窗口不会失去目录身份；双开一次打开当前项目的 Claude + Codex 并自动排好。

  • CLI 输出雷达——哪个终端正在产生输出，标题栏和侧栏立即亮起波形；Relay 只观察字节流，不解析、不保存内容
  • 待看队列与行动路由——后台终端的新输出沉淀为待看标记，⌥⌘J 按最早顺序逐个带回，⌥⌘K 一步返回来源
  • CLI 桌面记忆——重启后先显示上次布局的缩略图，由你决定恢复还是忘记，不会擅自启动任何 CLI
02 · 串联方式

五种把智能体连起来的方法

⇄

圆桌对话

2–4 个智能体按座次轮转交谈，各自保持独立会话记忆；你可以随时以主持人身份插话，或把各方立场导出成纪要。

⋈

同时发送

同一提示词并行交给多个智能体对照回答；worktree 隔离模式下各自改代码，按文件、按改动块选择性采纳。

›

顺序接力

2–4 个步骤在 daemon 中按序接力，上一步的答案交给下一步；关掉窗口串联继续跑，还能从任意一环分叉出新链。

⚡

结果裁决

把多个终端的画面冻结成对照卡片，交给另一个 CLI 或 daemon 裁判评判；结论、理由、把握程度结构化返回。

⑂

上下文分叉

截取任一终端的活动画面，带上你的继续指令填入其他 CLI——bracketed paste 只填入、不执行，回车永远在你手里。

03 · 决策链

每个结论都带着完整证据

从冻结现场到最终裁决，每一步都是显式操作、只读证据：Relay 不替你按回车，不偷偷改写记录，也不自动评价结果。保存的检查点带着当时的 git 基线，随时可以检查代码漂移、对比派生分支。

冻结现场只读快照→ 结果裁决人握回车→ 封存结果显式点击→ 私有检查点git 基线→ 血缘 / 漂移多代对比
04 · 隐私边界

全部只发生在你的 Mac 上

LOCAL ONLY

Relay 自身没有任何远端服务：没有账号、没有遥测、没有云同步。终端画面快照只存在于当前进程内存，不进剪贴板、不写磁盘；嵌入式终端的 PTY 内容从不记录。只有你显式保存的决策检查点会落盘——目录 0700、文件 0600，删除时进废纸篓而不是直接销毁。

05 · 开放接入

一份 manifest，接入任何 CLI

按行输出的 CLI 走 generic 段，讲 ACP（Agent Client Protocol）的走 acp 段——零代码。声明会话参数即可获得续聊，应用内置导入校验和健康检查。

// adapters/manifests/examples/echo.json
{
  "id": "echo",
  "display_name": "Echo CLI",
  "requirements": [{ "env": "ECHO_BIN", "candidates": ["echo"] }],
  "generic": {
    "command": "ECHO_BIN",
    "resume_arguments": ["--resume", "{session}"]
  }
}
06 · 上手

两条命令构建安装

# 依赖：macOS 14+ · Swift 5.9+ · Rust · Node.js
git clone https://github.com/shinteni/relay-workbench.git
cd relay-workbench
./scripts/package-macos-app.sh        # 构建并打包 dist/Relay.app
ditto dist/Relay.app ~/Applications/  # 安装，双击即用
